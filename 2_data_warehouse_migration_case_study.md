# Data Warehouse & Migration Integrity (DPDS)

## Overview
This project focused on validating data across legacy systems, Data Warehouse layers, and new cloud migration targets to ensure consistency during a large-scale enterprise migration.

## My Role
Test Lead Analyst overseeing data quality validation and ETL integrity checks.

## Tools & Technologies
- SQL (Oracle, Snowflake)
- Python
- Tableau
- Excel

## Responsibilities
- Performed source-to-target SQL comparisons for millions of records.
- Validated ETL logic and transformation rules.
- Ensured data governance alignment across multiple business domains.
- Supported DWH architecture teams by verifying schema and mapping accuracy.

## Business Impact
- Prevented data inconsistencies during migration.
- Ensured stable and reliable reporting across new systems.
- Strengthened enterprise-wide data governance.

## Technical Highlights
- Reconciliation scripts for row counts, key checks, and business rules.
- Regression testing of ETL pipelines.
- Validation of slowly changing dimensions and fact table structures.
